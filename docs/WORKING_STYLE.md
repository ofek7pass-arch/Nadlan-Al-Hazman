# סגנון עבודה — דרישות מאופק (Claude)

## עצמאות בזיהוי חסמים טכניים

**הדרישה:** כאשר סריקה נכשלת או כשיש אינדיקציה לחסימת IP / צורך ב-API מורכב — לא להמתין לפני שהמשתמש מתלונן. לזהות בזמן, לנתח, ולהציג המלצה יזומה.

**למה:** בפרויקט נדל"ן, הצגנו scrapers של יד2 ומדלן שהחזירו 0 תוצאות לאורך זמן. ה-debug הראה שהבעיה היא חסימת IP מ-Railway US West — דבר שהיה ניתן לצפות מראש. הבעיה התגלתה רק לאחר זמן רב של ניסיונות, במקום שתועלה יזומה מוקדם.

**איך ליישם:**
1. כשסריקה מחזירה 0 תוצאות פעמיים ברצף — לעצור ולאבחן לפני שממשיכים
2. כשבונים scraper לאתר ישראלי שרץ על שרת זר — לציין מראש את סיכון החסימה
3. לנתח אפשרויות (proxy / שרת ישראלי / API רשמי) ולהמליץ לפני שמבזבזים זמן על ניסיונות נוספים
4. להציג: "זו בעיה תשתיתית, לא קוד — הנה האפשרויות שלנו"

---

## פורמט המלצה ליזמה בעת חסימה

כשמזוהה חסם טכני מהותי:

1. **הגדרת הבעיה** — מה בדיוק חסום ולמה (IP blocking / auth / rate limit)
2. **אפשרויות פתרון** — עם עלות, מורכבות ואמינות לכל אחת
3. **המלצה** — מה עדיף ולמה
4. **מה לא נדרש** — מה לא כדאי לנסות (חוסך זמן)
